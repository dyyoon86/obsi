# Shopping Shorts Tool Validation via Direct Operation

Captured from a session analyzing Korean/YouTube shopping-shorts videos and competitor accounts for a future tool/SaaS direction.

## Core lesson

If the user wants to sell tools to people doing shopping shorts / Coupang Partners / affiliate short-form commerce, do **not** jump straight to building or selling the tool. The user correctly identified that credibility requires first operating an account and producing evidence.

Preferred sequence:

1. Directly operate a shopping-shorts account.
2. Use the agent to help with sourcing, hooks, scripts, CTAs, and analysis.
3. Track the actual repetitive pain points and performance data.
4. Build small internal tools for the operator first.
5. Sell the tool later as "the system I used while running this account," backed by data.

Positioning logic: in a gold rush, selling shovels is attractive, but a shovel seller needs proof the shovel works in that mine.

## Important correction from user

The user rejected intuition-based product picking. For this workflow, product candidates must come from evidence:

- Douyin/TikTok/Taobao hot-item signals
- Korean shopping-shorts competitor accounts
- High-view videos and repeated product patterns
- Comments showing purchase intent ("어디서 사요?", "링크?", "제품명?")
- Coupang availability / similar product availability

Do **not** propose products purely from general common sense. If suggesting starter products, label them as unvalidated hypotheses unless they came from observed competitor/hot-item evidence.

## First practical sourcing path

For beginners, start with Korean shopping-shorts account reverse-engineering before trying to discover products from Douyin cold search.

Workflow:

1. Collect Korean shopping-shorts / 생활꿀템 / 살림템 / 주방템 YouTube Shorts, TikTok, or Instagram accounts.
2. Pull recent/high-view videos.
3. Identify product category, hook pattern, and visible product.
4. Check comments for purchase intent.
5. Check Coupang for similar products, price range, reviews, Rocket delivery if relevant.
6. Score candidates and only produce videos for high-scoring items.

## Observed hook patterns

Across analyzed accounts/videos:

- Product name is usually hidden in the title; curiosity leads.
- Frequent wording: "이걸", "이거", "붙여보세요", "깔아보세요", "세탁기에 붙인다고?", "아는 사람만", "다이소에도 없는", "일본/미국에서 대박".
- Relationship/social proof frames work for household audiences: "시누이네서 보고", "결벽증 언니가 추천", "엄마한테 혼날 뻔".
- Problem/after-state products work well: cleaning, storage, repair, self-interior, laundry, bathroom, kitchen, monsoon/summer prep.

## Candidate scoring rubric

Use a 100-point rubric before producing:

- Viral evidence: 30
  - High-view source video, repeated product across accounts, purchase-intent comments.
- Coupang conversion potential: 25
  - Similar product exists, reasonable price, reviews/sales proof, low purchase friction.
- Video demonstration strength: 20
  - Feature visible in 1 second, clear before/after or problem/solution, can fit 15 seconds.
- Target fit: 15
  - Fits Korean 40–60 household/salim audience or chosen niche.
- Risk: 10
  - Low brand/IP dependence, low health/medical claim risk, manageable reuse risk.

Default: only produce candidates scoring 70+.

## Internal tools to build before external SaaS

Start as operator-assist tools, not public SaaS:

1. Hot-item validation board
   - Input: source video URL/screenshot/product guess.
   - Output: product name candidates, Korean/Coupang search keywords, Chinese/English source keywords, hook candidates, score, risks.
2. Script/CTA generator
   - Input: validated product + target + platform.
   - Output: 1.5s hooks, 15s scripts, cut list, caption, comments/DM CTA, Coupang disclosure text.
3. Performance tracker
   - Record product, hook, CTA, platform, views, comments, profile visits, link clicks, Coupang clicks/revenue.

## Evidence threshold before selling

The user does not need 월300 proof before selling, but needs operating evidence. Possible thresholds:

- 14 days / 42 posts with views, clicks, and at least some conversion signal.
- 30 days / ~100 posts with repeated winning categories and real tracking data.
- Stronger: recurring link clicks/revenue and proof that internal tools reduced production time.

Avoid selling as "AI tool that makes you rich." Sell as "the product-sourcing/script system built while operating a shopping-shorts account."
