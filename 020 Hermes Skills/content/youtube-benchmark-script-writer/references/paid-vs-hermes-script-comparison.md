# Paid Script vs Hermes Script Comparison — VoiceBox/HyperFrames

## Context

The user compared a paid script generated from the same benchmark/target-video method against the Hermes-produced 투두TV script package for the VoiceBox/HyperFrames video.

## Key finding

The paid script is stronger as a polished, high-concept narrated tutorial. The Hermes script is stronger as a 투두TV-native production package: face-cam, lived-use tone, caveats, long-script chunking workflow, ethics, and asset-sharing CTA.

Best future output should combine both:

- Paid script strengths: cinematic promise, tighter A-to-Z pipeline, stronger tool positioning, orchestration framing, design iteration stage.
- Hermes strengths: authenticity, direct test/result framing, realistic limitations, chunking workflow, face-cam/demo directing, free asset package.

## Paid script strengths to absorb

1. **High-concept promise in hook**
   - `단 한 줄의 코드도 짜지 않고, 한마디 녹음도 없이, AI와 대화만으로...`
   - Stronger than a purely cautious hook.
   - Use it, but avoid overclaiming; pair with `직접 해보니 긴 대본은 쪼개야 안정적이었다`.

2. **Clear pipeline naming**
   - `보이스박스 설치와 목소리 복제 → 하이퍼프레임으로 모션그래픽 자동 생성 → 음성과 영상 결합 → AI와 대화하며 디자인 수정`
   - Future scripts should name the pipeline stages explicitly.

3. **Tool-context authority**
   - Explains why VoiceBox matters: local processing, free alternative to paid voice tools, privacy angle.
   - Explains why HyperFrames matters: agent-native video, HTML/CSS/JS timeline, AI edits structure.
   - Only include specific claims (dates, creators, stars, licensing) when verified.

4. **Orchestrator metaphor**
   - Claude Code as `지휘자/오케스트레이터` connecting tools.
   - This makes multi-tool automation easier to understand.

5. **Design iteration stage**
   - After first result, ask Claude to modify tone, color, font, layout, and rerender.
   - This is highly visual and should be a standard demo beat for AI production scripts.

6. **Future episode bridge**
   - Ends by previewing the next automation extension.
   - Good for series retention.

## Hermes script strengths to keep

1. **투두TV fit**
   - First-person `제가 직접 해봤습니다` tone.
   - Stronger channel identity than polished generic narration.

2. **Reality and trust**
   - VoiceBox quality is good, but long scripts can fail.
   - Chunking into 20–40 second segments is a real practical insight and should stay central.

3. **Production package**
   - Face-cam directing, screen-recording cues, pinned comment, description, asset checklist.
   - The user values all-in-one packages.

4. **Ethics**
   - Voice cloning requires explicit consent / own voice only.
   - Must not be omitted.

5. **Reusable assets CTA**
   - Installation checklist, Claude Code prompt, chunking template, ethics checklist.

## Updated future synthesis rule

For VoiceBox/HyperFrames-like scripts, write in this hybrid structure:

1. Result-first hook with a bold promise. **Do not put downsides/limitations in the hook.** The hook's job is to make viewers keep watching.
2. Face-cam trust reset with curiosity intact: `제가 직접 해봤고, 전체 과정을 보여드리겠습니다` rather than immediately saying `막히는 점이 있었다`.
3. Pipeline preview with named stages.
4. Tool 1: why it matters + demo.
5. Tool 2: why it matters + demo.
6. Orchestrator step: Claude Code connects voice + video.
7. Design iteration: natural-language visual revision.
8. Practical limitation / reality check: long scripts must be chunked 20–40 seconds. Put this in the mid/late body, description, pinned comment, or checklist — not in the hook.
9. Final evaluation: best uses / not good uses.
10. CTA: share checklist/prompts/templates and ask for errors/use-cases.

## Avoid

- Overclaiming `비용 0원` if Claude Code subscription or hardware/time is required.
- Unverified facts like dates, GitHub stars, creator names, licensing terms.
- Sounding like a generic AI narrator with no lived experience.
- Omitting the long-script chunking insight.
